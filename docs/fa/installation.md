# راهنمای نصب

این راهنما نحوه استقرار **PD-Element Local** روی یک سرور Ubuntu تازه نصب‌شده را توضیح می‌دهد.

نصب‌کننده کاملاً تعاملی (Interactive) است و شما را در تمام مراحل پیکربندی موردنیاز راهنمایی می‌کند.

---

# قبل از شروع

پیش از اجرای نصب‌کننده، اطمینان حاصل کنید که پیش‌نیازهای زیر فراهم شده‌اند.

## سیستم‌عامل

نسخه‌های پشتیبانی‌شده:

- Ubuntu Server 22.04 LTS
- Ubuntu Server 24.04 LTS

به‌شدت توصیه می‌شود از یک نصب کاملاً تمیز (Clean Installation) استفاده کنید.

---

## نیازمندی‌های سخت‌افزاری

### استقرار کوچک (Small Deployment)

- 2 هسته پردازنده (CPU)
- 2 گیگابایت حافظه RAM
- 20 گیگابایت فضای ذخیره‌سازی

مناسب برای:

- Home Labs
- دفاتر کوچک
- محیط‌های توسعه (Development)

---

### استقرار متوسط (Medium Deployment)

- 4 هسته پردازنده (CPU)
- 8 گیگابایت حافظه RAM
- فضای ذخیره‌سازی SSD

مناسب برای:

- شرکت‌ها
- مدارس
- سازمان‌های کوچک

---

### استقرار بزرگ (Large Deployment)

- 8 هسته پردازنده یا بیشتر
- 16 گیگابایت RAM یا بیشتر
- فضای ذخیره‌سازی SSD/NVMe

مناسب برای:

- محیط‌های Enterprise
- سازمان‌های بزرگ

---

# نیازمندی‌های شبکه

سرور باید دارای موارد زیر باشد:

- آدرس IP ثابت (Static LAN IP)
- DNS داخلی
- دسترسی به اینترنت در زمان نصب (توصیه می‌شود)

پس از پایان نصب، برای عملکرد عادی سیستم دیگر نیازی به دسترسی اینترنت نخواهد بود.

---

# پورت‌های موردنیاز

پورت‌های زیر را روی فایروال باز کنید.

| پورت | پروتکل | کاربرد |
|-------|----------|---------|
| 80 | TCP | HTTP |
| 443 | TCP | HTTPS |
| 3478 | UDP/TCP | TURN |
| 5349 | TCP | TURN TLS |

---

# DNS داخلی

پیش از نصب، رکوردهای DNS را ایجاد کنید.

مثال:

| نام میزبان (Hostname) | آدرس IP |
|----------|------------|
| matrix.company.local | 192.168.10.5 |
| element.company.local | 192.168.10.5 |

پیش از ادامه، صحت عملکرد DNS را بررسی کنید.

مثال:

```bash
nslookup matrix.company.local
```

یا

```bash
ping matrix.company.local
```

هر دو نام میزبان باید به آدرس IP سرور اشاره کنند.

# اجرای نصب‌کننده

دستور زیر را اجرا کنید:


نصب‌کننده باید با **root** یا **sudo** اجرا شود.

# نصب‌کننده تعاملی (Installation Wizard)

در طول نصب، چند سؤال از شما پرسیده خواهد شد.

---

## Matrix Homeserver

مثال:

```
matrix.company.local
```

این آدرس سرور Matrix شما است.

---

## دامنه Element

مثال:

```
element.company.local
```

این همان آدرسی است که کاربران برای دسترسی به رابط وب استفاده خواهند کرد.

---

## دامنه اصلی (Base Domain)

مثال:

```
company.local
```

این دامنه به‌صورت داخلی برای تولید گواهی‌ها استفاده می‌شود.

---

## آدرس IP سرور

مثال:

```
192.168.10.5
```

این مقدار باید آدرس IP ثابت سرور در شبکه داخلی باشد.

---

## مسیر ذخیره‌سازی PKI

مقدار پیش‌فرض:

```
/etc/pd-element-local/pki
```

این مسیر موارد زیر را نگهداری می‌کند:

- Root CA
- Intermediate CA
- گواهی‌ها (Certificates)
- کلیدهای خصوصی (Private Keys)

---

## مدت اعتبار Root CA

مثال:

```
20 years
```

Root CA معمولاً به‌ندرت تغییر می‌کند.

---

## مدت اعتبار گواهی سرور

مثال:

```
730 days
```

اعتبار طولانی‌تر، نیاز به نگهداری را کاهش می‌دهد و همچنان قابل مدیریت باقی می‌ماند.

---

## SANهای اضافی

اختیاری.

مثال:

```
chat.company.local
turn.company.local
```

در صورتی که به آن‌ها نیاز ندارید، این بخش را خالی بگذارید.

# در طول نصب چه اتفاقی می‌افتد؟

نصب‌کننده به‌صورت خودکار وظایف زیر را انجام می‌دهد:

- نصب Matrix Synapse
- نصب Element Web
- نصب Coturn
- پیکربندی Nginx
- ایجاد Enterprise PKI
- ایجاد Root CA
- ایجاد Intermediate CA
- صدور گواهی‌های سرور
- پیکربندی TLS
- پیکربندی TURN
- پیکربندی Matrix
- پیکربندی Element
- فعال‌سازی سرویس‌ها
- اجرای بررسی سلامت (Health Checks)

در حالت عادی، هیچ پیکربندی دستی موردنیاز نیست.

---

# پس از نصب

پس از اتمام نصب:

1. یک Health Check اجرا کنید.
2. گواهی Root CA را خروجی بگیرید.
3. Root CA را روی تمام دستگاه‌های کاربران نصب کنید.
4. یک حساب Administrator ایجاد کنید.
5. ثبت‌نام عمومی (Public Registration) را غیرفعال کنید (توصیه می‌شود).
6. حساب‌های کاربران را ایجاد کنید.
7. اولین نسخه پشتیبان را ایجاد کنید.

---

# اولین ورود

آدرس زیر را باز کنید:

```
https://element.company.local
```

با استفاده از حساب Administrator که در زمان نصب ایجاد کرده‌اید وارد شوید.

# به‌روزرسانی

برای به‌روزرسانی Element یا انجام عملیات نگهداری، به‌جای جایگزین کردن دستی فایل‌ها، از گزینه‌های موجود در منوی اصلی استفاده کنید.

# عیب‌یابی

اگر نصب‌کننده خطایی گزارش کرد:

1. صحت عملکرد DNS را بررسی کنید.
2. قوانین فایروال را بررسی کنید.
3. زمان و تاریخ سرور را بررسی کنید.
4. Health Check را اجرا کنید.
5. Fix Wizard را اجرا کنید.
6. لاگ‌های تولیدشده را بررسی کنید.

بیشتر مشکلات رایج بدون نیاز به نصب مجدد سرور قابل برطرف شدن هستند.
