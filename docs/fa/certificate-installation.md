# نصب Root CA روی دستگاه‌های کاربران

پس از نصب PD-Element Local، دستگاه‌های کاربران باید به Root Certificate Authority (Root CA) سازمان شما اعتماد داشته باشند.

بدون نصب Root CA، مرورگرها و کلاینت‌های Matrix هشدارهای امنیتی نمایش خواهند داد، زیرا گواهی‌های شما توسط زیرساخت کلید عمومی (PKI) اختصاصی خودتان صادر شده‌اند، نه توسط یک مرجع صدور گواهی عمومی مانند Let's Encrypt.

این راهنما نحوه نصب Root CA را روی سیستم‌عامل‌های پشتیبانی‌شده توضیح می‌دهد.

---

# خروجی گرفتن از Root CA

قبل از نصب گواهی روی دستگاه‌های کاربران، Root CA را از منوی **PKI Management** خروجی بگیرید.

منوی اصلی

↓

PKI Management

↓

Export Root CA

فایل خروجی معمولاً با نام زیر است:

```
root-ca.crt
```

این فایل را به‌صورت امن در اختیار کاربران خود قرار دهید.

---

# ویندوز

## نسخه‌های پشتیبانی‌شده

- Windows 10
- Windows 11
- Windows Server

## نصب

1. روی گواهی دوبار کلیک کنید.
2. روی **Install Certificate** کلیک کنید.
3. گزینه زیر را انتخاب کنید:

```
Local Machine
```

4. روی **Next** کلیک کنید.
5. گزینه زیر را انتخاب کنید:

```
Place all certificates in the following store
```

6. گزینه زیر را انتخاب کنید:

```
Trusted Root Certification Authorities
```

7. مراحل نصب را تا پایان ادامه دهید.

برای انجام این عملیات، دسترسی Administrator لازم است.

---

## بررسی نصب

برنامه زیر را باز کنید:

```
certmgr.msc
```

به مسیر زیر بروید:

```
Trusted Root Certification Authorities

↓

Certificates
```

Root CA شما باید در این فهرست نمایش داده شود.

---

# Ubuntu / Debian

گواهی را کپی کنید.

```
sudo cp root-ca.crt /usr/local/share/ca-certificates/
```

سپس مخزن گواهی‌ها را به‌روزرسانی کنید.

```
sudo update-ca-certificates
```

خروجی مورد انتظار:

```
1 added, 0 removed
```

---

# RHEL / Rocky / AlmaLinux

گواهی را کپی کنید:

```
sudo cp root-ca.crt /etc/pki/ca-trust/source/anchors/
```

سپس دستور زیر را اجرا کنید:

```
sudo update-ca-trust
```

---

# Arch Linux

گواهی را کپی کنید:

```
sudo cp root-ca.crt /etc/ca-certificates/trust-source/anchors/
```

سپس دستور زیر را اجرا کنید:

```
sudo trust extract-compat
```

---

# macOS

برنامه زیر را باز کنید:

```
Keychain Access
```

فایل زیر را وارد (Import) کنید:

```
root-ca.crt
```

آن را به محل زیر منتقل کنید:

```
System Keychain
```

روی گواهی دوبار کلیک کنید.

بخش زیر را باز کنید:

```
Trust
```

گزینه زیر را تنظیم کنید:

```
When using this certificate:

Always Trust
```

پنجره را ببندید.

رمز عبور Administrator را وارد کنید.

```

```


در صورت فعال نکردن Trust، Safari و سایر برنامه‌ها همچنان هشدارهای مربوط به گواهی را نمایش خواهند داد.

# Element Web

اگر مرورگر به Root CA اعتماد داشته باشد، Element Web نیز به‌صورت خودکار به گواهی‌های سرور Matrix شما اعتماد خواهد کرد.

---

# بررسی صحت نصب

آدرس Element Web خود را باز کنید.

مثال:

```
https://element.company.local
```

مرورگر باید موارد زیر را نمایش دهد:

- اتصال امن (Secure Connection)
- گواهی معتبر (Valid Certificate)
- بدون هشدار امنیتی (No Security Warnings)

---

# مشکلات رایج

## گواهی مورد اعتماد نیست

دلایل احتمالی:

- گواهی در مخزن اشتباه نصب شده است.
- دسترسی Administrator وجود ندارد.
- مرورگر از مخزن گواهی اختصاصی خود استفاده می‌کند.
- گواهی اشتباه صادر یا خروجی گرفته شده است.

---

## هشدار گواهی همچنان نمایش داده می‌شود

موارد زیر را بررسی کنید:

- Root CA به‌درستی نصب شده باشد.
- زنجیره گواهی (Certificate Chain) صحیح باشد.
- گواهی سرور با نام میزبان (Hostname) مطابقت داشته باشد.
- فیلدهای SAN شامل نام میزبان درخواستی باشند.

---

## دستگاه‌های همراه همچنان هشدار نمایش می‌دهند

موارد زیر را بررسی کنید:

- Root CA نصب شده باشد.
- Trust در iOS فعال شده باشد.
- گواهی در اندروید به‌درستی نصب شده باشد.
- زمان و تاریخ دستگاه صحیح باشد.

---

# توصیه‌های امنیتی

- هرگز کلید خصوصی Root CA را توزیع نکنید.
- فقط گواهی عمومی Root CA را در اختیار کاربران قرار دهید.
- تا حد امکان کلید خصوصی Root CA را به‌صورت آفلاین نگهداری کنید.
- از نسخه‌های پشتیبان PKI محافظت کنید.
- در صورت افشای کلید خصوصی، گواهی‌ها را فوراً ابطال (Revoke) کنید.
- تاریخ انقضای گواهی‌ها را به‌صورت منظم بررسی کنید.

---

# مرحله بعد

پس از اینکه تمام دستگاه‌های کاربران به Root CA اعتماد کردند:

1. حساب‌های Administrator را ایجاد کنید.
2. حساب‌های کاربران را ایجاد کنید.
3. پیام‌رسانی را آزمایش کنید.
4. تماس صوتی را آزمایش کنید.
5. تماس تصویری را آزمایش کنید.
6. اولین نسخه پشتیبان را ایجاد کنید.

اکنون پلتفرم ارتباطی خصوصی Matrix شما آماده بهره‌برداری در محیط عملیاتی (Production) است.
